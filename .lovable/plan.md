# إعادة تصميم صفحة رفع المستندات

تطبيق التصميم الجديد المرسل من العميل على صفحة `/` (الرئيسية) فقط، بدون تغيير الـ logic أو الـ backend أو التراجم. كل المعلومات لسا بتنحفظ على سيرفر Directus داخل الإمارات.

## شو راح يتغير بصرياً

### قبل (الحالي)
- 3 بطاقات كبيرة (Registration / Emirates / License) كلٌ عبارة عن مربع dashed كبير.
- تحتها 3 بطاقات إضافية بنفس الحجم (مركبة، فحص، مرفقات) ظاهرين دايماً.
- صف KYC قبل المستندات.

### بعد (المطلوب من الموكاب)
- **قائمة صفوف أفقية compact** لكل مستند: أيقونة معاينة (يسار) + اسم المستند + شارة "مطلوب" أحمر + زر كاميرا أخضر (يمين).
- **قسم "المستندات الأساسية"** بعنوان وdot أخضر، فيه 3 صفوف: الملكية، الرخصة، الهوية.
- **قسم "مستندات إضافية (اختياري)"** قابل للطي (accordion):
  - مغلق افتراضياً → بس زر "+" دائري + نص "إضافة مستندات إضافية".
  - مفتوح → يظهر صفوف للمركبة، الحيازة، عقد/اتفاقية، مستند آخر — كل صف فيه زر سلة 🗑️ أحمر للحذف.
  - زر "+ إضافة مستند آخر" أسفل القسم.
- **زر "إرسال المستندات"** أخضر كبير مع أيقونة طائرة ورقية.
- **3 شارات ثقة** أسفل الزر بشكل أفقي: ⏱ سريع → 🛡 آمن → ✓ موثوق (مع أسهم بينهم).
- **KYC** ينقل لقسم منفصل تحت زر الإرسال (يظهر بس لما يضيف المستخدم اللينك التواصلي/الدفع — حسب الموكاب الجزء السفلي).

## التفاصيل التقنية

### ملف جديد: `src/components/DocumentRow.tsx`
- Props: `icon`, `label`, `required`, `file/files`, `onChange`, `onRemove?`, `accept`, `allowVideo?`
- الشكل: `<div>` flex بين الأيقونة (يمين بـ RTL) + label + status (أحمر "مطلوب" أو سلة حذف) + زر كاميرا أخضر مربّع.
- لما يكون فيه ملف مرفوع: الأيقونة تتحول لـ thumbnail صغير + checkmark أخضر بدل "مطلوب".
- يستخدم نفس الـ validation logic من `UploadCard` (5MB، JPG/PNG/PDF).

### ملف جديد: `src/components/OptionalDocsSection.tsx`
- Accordion يبدأ مغلق، فيه:
  - حالة مغلقة: بطاقة dashed فيها `+` و"إضافة مستندات إضافية" + النص الإرشادي "مثل: صورة المركبة، صورة الحيازة، عقد، وغيرها".
  - حالة مفتوحة: قائمة من `DocumentRow` لكل من: مركبة، حيازة، عقد، مستند آخر (كلٌ بأيقونة من lucide-react: Car, Home, FileText, FolderOpen).
  - زر "+ إضافة مستند آخر" تحت القائمة لإضافة slot ديناميكي جديد (للملفات الحرة).

### تعديل: `src/routes/index.tsx`
- استبدال البلوك `{/* Required documents */}` و`{/* Optional uploads */}` و`{/* Other attachments */}` بالمكونات الجديدة.
- نقل بلوك KYC تحت زر الإرسال (لتطابق الموكاب الي يظهر "لإرسال رابط الدفع" أسفل الصفحة).
- إضافة بلوك الـ trust badges الـ 3 الأفقية تحت زر الإرسال (موجود جزئياً في `t.hero` — استخدام نفس النصوص).
- ما في تغيير على `submitUpload()` أو الـ state أو الـ validation.

### تعديل: `src/components/UploadCard.tsx` و `MultiUploadCard.tsx`
- **بدون أي تغيير**. تظل موجودة (مستخدمة في صفحة `/r/$requestId` لإعادة الرفع).

## الترجمات
- ما في حاجة لإضافة مفاتيح جديدة — كل النصوص الظاهرة بالموكاب موجودة في `translations.ts`:
  - "المستندات الأساسية" = `t.upload.cards.*` (موجود)
  - "مطلوب" = نضيف مفتاح واحد `t.upload.required` لو مش موجود
  - "اختياري" = `t.upload.optional` (موجود)
  - شارات الثقة = `t.hero.*` (موجود)

## ملاحظات مهمة
- **نفس الـ backend**: كل البيانات لسا بترجع لسيرفر Directus داخل الإمارات. ما في أي تغيير على `submitUpload()` أو الـ proxy.
- **نفس الـ validation**: 5MB حد أقصى، JPG/PNG/PDF فقط للأساسية، فيديو حتى 50MB للمركبة.
- **Responsive**: التصميم في الموكاب موبايل-فيرست؛ على الشاشات الكبيرة بنوسّع الـ container ونحافظ على نفس بنية الصفوف (مش grid).
- **بدون طي/إخفاء KYC الموجود**: نخليه ظاهر تحت زر الإرسال زي ما الموكاب يقول "لإرسال رابط الدفع، أدخل بياناتك".
